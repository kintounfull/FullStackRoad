            截取返回键

            在Activity和Fragment中，物理返回键默认是不会作用于WebView上的，需要让WebView截取才能有效。

            // WebView.java
@Override
public boolean onKeyUp(int keyCode, KeyEvent event) {
    // 网页截取物理返回键进行回退操作
    if (mAllowGoBack
     && keyCode == KeyEvent.KEYCODE_BACK
     && canGoBack()) {
        goBack();
        return true;
    }
    return super.onKeyUp(keyCode, event);
}

            Google Play应用商店审核要求

            Google Play要求必须在WebView SSL错误时弹窗提示用户，由用户进行选择处理，否则无法通过审核上线发布。

            如下重载实现，重载里不能调用super.onReceivedSslError，因为默认是调用handler.cancel()取消。

            mWebView.setWebViewClient(new WebViewClientCompat() {
    /**
     * Notify the host application that an SSL error occurred while loading a resource.
     *
     * @param view The WebView that is initiating the callback.
     * @param handler An {@link SslErrorHandler} that will handle the user's  response.
     * @param error The SSL error object.
     */
    @Override
    public void onReceivedSslError(WebView view, SslErrorHandler handler, SslError error) {
        QLog.w(TAG, "onReceivedSslError", error);

        // Google Play应用市场提交应用审核必现Alert弹框提示用户并进行相应处理，否则会审核不通过
        ((BaseActivity)view.getContext()).showAlert(R.string.ssl_cert_fail_tip,
                R.string.button_yes,
                new View.OnClickListener() {
                    @Override
                    public void onClick(View v) {
                        handler.proceed();
                    }
                },
                R.string.button_no,
                new View.OnClickListener() {
                    @Override
                    public void onClick(View v) {
                        handler.cancel();
                    }
                });
    }
});

            内存泄漏

            在Android 5.1上（没有关注后续系统版本是否解决？），WebView退出后极有可能导致Activity无法释放，造成内存泄漏。

            问题原因

            从上图中可以看出，调用关系是BookDetailActivity->BaseWebView->AwContents->mComponentCallbacks，而TBReaderApplication中的 mComponentCallbacks（它是一个array list）里面会持有activity实例。这个mComponentCallbacks的注册和去注册是在AwContents的onAttachedToWindow和onDetachedFromWindow中，但Android 5.1在onDetachedFromWindow的最前面判断如果isDestroyed()则直接返回，造成走不到去注册的地方，而WebView的destroy有可能在onDetachedFromWindow前调用，导致mComponentCallbacks会一直持有activity实例，导致activity无法释放

            • org.chromium.android_webview.AwContents

              @Override
public void onAttachedToWindow() {
    if (isDestroyed()) return;
    if (mIsAttachedToWindow) {
        Log.w(TAG, "onAttachedToWindow called when already attached. Ignoring");
        return;
    }
    mIsAttachedToWindow = true;

    mContentViewCore.onAttachedToWindow();
    nativeOnAttachedToWindow(mNativeAwContents, mContainerView.getWidth(),
            mContainerView.getHeight());
    updateHardwareAcceleratedFeaturesToggle();

    if (mComponentCallbacks != null) return;
    mComponentCallbacks = new AwComponentCallbacks();
    mContext.registerComponentCallbacks(mComponentCallbacks);
}

@Override
public void onDetachedFromWindow() {
    if (isDestroyed()) return;
    if (!mIsAttachedToWindow) {
        Log.w(TAG, "onDetachedFromWindow called when already detached. Ignoring");
        return;
    }
    mIsAttachedToWindow = false;
    hideAutofillPopup();
    nativeOnDetachedFromWindow(mNativeAwContents);

    mContentViewCore.onDetachedFromWindow();
    updateHardwareAcceleratedFeaturesToggle();

    if (mComponentCallbacks != null) {
        mContext.unregisterComponentCallbacks(mComponentCallbacks);
        mComponentCallbacks = null;
    }

    mScrollAccessibilityHelper.removePostedCallbacks();
}

            解决办法

            如下在WebView中重载WebView.destroy()方法，或者在Activity/Fragment销毁时，进行如下相同的操作。

            @Override
public void destroy() {
    mIsDestroy = true;

    // 规避可能的内存泄漏
    ViewParent parent = getParent();
    if (parent != null) {
        ((ViewGroup)parent).removeView(this);
    }

    stopLoading();
    
    // 退出时调用此方法，移除绑定的服务，否则某些特定系统会报错
    getSettings().setJavaScriptEnabled(false);
    
    clearView();
    removeAllViews();
    
    // 是否是必要项？
    clearHistory();

    super.destroy();
}

            支持JS打开新窗口

            JS的window.open(URL,NAME,...)和HTML的链接<a href=URL target=NAME>用于打开一个新的浏览器窗口或查找一个已命名的窗口。

            NAME，可选。指定target属性或窗口的名称。支持以下值：

            • _blank ，URL加载到一个新的窗口。window.open()的默认值
            • _self ，URL加载替换当前页面。<a>的默认值
            • _parent，URL加载到父框架
            • _top ，清空头部框架内所有页面，并加载URL

            Android默认不支持**_blank**操作，需要进行以下处理：

            mWebView.getSettings().setJavaScriptCanOpenWindowsAutomatically(true);
mWebView.getSettings().setSupportMultipleWindows(true);

// 打开新窗口加载URL
mWebView.setWebChromeClient(new WebChromeClient() {
    @Override
    public boolean onCreateWindow(WebView view, boolean isDialog, boolean isUserGesture, Message resultMsg) {
        final Context context = view.getContext();
        SafeWebView xView = new SafeWebView(context);
        xView.setWebViewClient(new WebViewClient() {
            @Override
            public boolean shouldOverrideUrlLoading(WebView view, String url) {
                Intent intent = new Intent(context, WebActivity.class);
                Bundle bundle = new Bundle();
                bundle.putString(WebActivity.SCHEMA_URL, url);
                intent.putExtras(bundle);
                context.startActivity(intent);
                return true;
            }
        });

        WebView.WebViewTransport transport = (WebView.WebViewTransport) resultMsg.obj;
        transport.setWebView(xView);
        resultMsg.sendToTarget();
        return true;
    }
});

// 将_blank改为同_self，当前窗口加载URL，有返回栈
mWebView.setWebChromeClient(new WebChromeClient() {
    @Override
    public boolean onCreateWindow(WebView webView, boolean isDialog, boolean isUserGesture, Message resultMsg) {
        SafeWebView xView = new SafeWebView(webView.getContext());
        xView.setWebViewClient(new WebViewClient() {
            @Override
            public boolean shouldOverrideUrlLoading(WebView view, String url) {
                // 此处可以是mWebView或webView，
                // 但不能是view这个WebView对象（xView），自己里再loadUrl会Crash
                webView.loadUrl(url);
                return true;
            }
        });

        WebView.WebViewTransport transport = (WebView.WebViewTransport) resultMsg.obj;
        transport.setWebView(xView);
        resultMsg.sendToTarget();
        return true;
    }
});

            集成/调用第三方WebView

            Android 4.4（API 级别 19）引入了基于 Chromium 的新版 WebView。此项变更会升级 WebView 的性能并标准化对 HTML5、CSS3 和 JavaScript 的支持，并可通过应用商店更新以与最新的网络浏览器保持一致。

            因此，在API 19以前，可通过APK集成第三方WebView或者调用系统中的其他WebView服务来提升WebView的性能和兼容性。
